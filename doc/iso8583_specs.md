# ISO 8583 メッセージ仕様書

## 1. 概要
ISO 8583は、カードを利用した金融取引（クレジットカード、デビットカード等）において、システム間でメッセージを交換するための国際標準規格です。メッセージは、要求 (Request)、応答 (Response)、通知 (Advice) などの種類に分かれ、取引の承認や清算に利用されます。

## 2. メッセージ構造
ISO 8583メッセージは、以下の3つの主要なパーツで構成されます。

### 2.1 MTI (Message Type Indicator)
メッセージの種類を定義する4桁の数値です。

| 桁 | 意味 | 主な値の内容 |
|---|---|---|
| 1 | ISO 8583の版 | 0:1987版, 1:1993版, 2:2003版, 9:私用 |
| 2 | メッセージクラス | 1:承認, 2:財務, 4:取消, 8:ネットワーク管理 |
| 3 | メッセージ機能 | 0:要求, 1:要求応答, 2:アドバイス, 4:通知 |
| 4 | メッセージ発生源 | 0:アクワイアラ, 1:アクワイアラ(リピート), 2:イシュア |

### 2.2 ビットマップ (Bitmap)
どのデータエレメント（フィールド）がメッセージに含まれているかを示すビット列です。
- **Primary Bitmap**: フィールド1〜64の有無を示します（必須）。
- **Secondary Bitmap**: フィールド65〜128の有無を示します（第1ビットが1の場合に存在）。
- **Third Bitmap**: フィールド129〜192の有無を示します（第65ビットが1の場合に存在）。

### 2.3 データエレメント (Data Elements)
実際の取引情報が含まれるフィールド群です。

## 3. データ型と長さの定義
各フィールドは、属性 (Attributes) と長さ (Length) によって定義されます。

| 記号 | 意味 |
|---|---|
| a | 英字 (Alpha) |
| n | 数字 (Numeric) |
| s | 特殊文字 (Special characters) |
| an | 英数字 (Alphanumeric) |
| ans | 英数字および特殊文字 (Alphanumeric and special characters) |
| b | バイナリデータ (Binary data) |
| z | トラック2, 3のデータ |

### 長さ指定
- **固定長**: 例 `n 6` (6桁の数字)
- **可変長 (Variable length)**:
    - `..` または `LLVAR`: 最大99文字（2桁の長さヘッダ）
    - `...` または `LLLVAR`: 最大999文字（3桁の長さヘッダ）

### 5. 属性の例
フィールドの属性と長さの組み合わせの表記例です。

| フィールド属性値 | 意味 |
|---|---|
| n6 | 6桁の固定長数字フィールド |
| n.6 | 最大6桁の可変長数字フィールド |
| a..11 | 最大11桁の可変長英字フィールド |
| b...999 | 999バイトのバイナリフィールド |
